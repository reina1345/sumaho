# プロンプトの基礎

## はじめに

プロンプトとは、AI（人工知能）に対して与える指示や質問のことです。適切なプロンプトを書くことで、AIから望ましい回答や成果物を得ることができます。この記事では、効果的なプロンプトを作成するための基礎知識を解説します。

## プロンプトとは

プロンプト（Prompt）は、AIモデルに対する入力テキストのことを指します。ChatGPTやClaude、その他の大規模言語モデル（LLM）とやり取りする際に、ユーザーが入力する文章がプロンプトです。

プロンプトの質によって、AIの応答の質が大きく変わります。これを「プロンプトエンジニアリング」と呼び、近年注目されているスキルの一つです。

## 良いプロンプトの特徴

効果的なプロンプトには、以下の特徴があります：

### 1. 明確性（Clarity）
- 何を求めているのかが明確
- 曖昧な表現を避ける
- 具体的な指示を含む

### 2. 具体性（Specificity）
- 詳細な情報を提供する
- 求める出力形式を指定する
- 必要な制約条件を明示する

### 3. 文脈の提供（Context）
- 背景情報を含める
- 目的や用途を説明する
- 対象読者を明示する

### 4. 構造化（Structure）
- 論理的な順序で情報を提示する
- 箇条書きや番号付きリストを活用する
- セクションに分けて整理する

## プロンプトの基本構成要素

効果的なプロンプトは、以下の要素で構成されます：

### 1. タスクの指定
AIに何をしてほしいのかを明確に伝えます。

```
例：「以下のテキストを要約してください」
例：「Pythonでフィボナッチ数列を生成する関数を書いてください」
```

### 2. コンテキスト（文脈）
タスクに関連する背景情報や前提条件を提供します。

```
例：「初心者向けのプログラミング講座で使用するために...」
例：「ビジネスメールの文脈で、フォーマルな表現を使って...」
```

### 3. 入力データ
処理対象となるデータやテキストを提供します。

```
例：「以下のテキスト: [ここにテキストを挿入]」
例：「この問題を解いてください: [問題文]」
```

### 4. 出力形式の指定
望ましい回答の形式や構造を指定します。

```
例：「箇条書きで3つのポイントにまとめてください」
例：「JSON形式で出力してください」
例：「500文字以内で説明してください」
```

### 5. 制約条件
遵守すべきルールや制限を明示します。

```
例：「専門用語を使わず、小学生にもわかる言葉で説明してください」
例：「コードにはコメントを含めてください」
例：「ポジティブなトーンで書いてください」
```

## 実践例

### 悪い例
```
プログラムを書いて
```

**問題点：**
- 何のプログラムか不明
- どの言語を使うのか不明
- どんな機能が必要か不明

### 良い例
```
Pythonで、ユーザーから数値を入力として受け取り、
その数値が素数かどうかを判定するプログラムを書いてください。

要件：
- 関数名は is_prime() とする
- 引数は整数を1つ受け取る
- 戻り値はブール値（True/False）
- エッジケース（0, 1, 負の数）も適切に処理する
- コメントを含め、コードを説明する
```

**優れている点：**
- 言語が明確（Python）
- 具体的な機能が定義されている
- 関数の仕様が詳細に記述されている
- エッジケースへの配慮が要求されている
- ドキュメントの必要性が明示されている

## プロンプト作成のベストプラクティス

### 1. 段階的に詳細化する
最初は簡潔なプロンプトから始め、必要に応じて詳細を追加していきます。

### 2. 例を示す（Few-shot Learning）
望ましい出力の例を1〜3個示すことで、AIの理解を助けます。

```
以下の形式で商品レビューを生成してください：

例1：
商品名: ワイヤレスイヤホン X100
評価: ★★★★☆
コメント: 音質が良く、バッテリーも長持ちします。

例2：
商品名: スマートウォッチ Y200
評価: ★★★★★
コメント: デザインが素晴らしく、健康管理機能も充実しています。

次の商品のレビューを生成してください：
商品名: ノートPC Z300
```

### 3. ロール（役割）を指定する
AIに特定の役割を与えることで、より適切な回答を得られます。

```
あなたは経験豊富なPythonプログラミング講師です。
初心者に変数の概念を説明してください。
```

### 4. 思考過程を求める
複雑な問題では、段階的な思考過程を示すよう求めます。

```
以下の問題を解く際に、ステップバイステップで考え方を示してください：
[問題文]
```

### 5. 制約を明確にする
避けてほしいことや、必須要件を明確に伝えます。

```
以下の条件を守って回答してください：
- 専門用語は使用しない
- 例を2つ以上含める
- 300文字以内にまとめる
```

### 6. 反復と改善
最初の結果が期待通りでなければ、プロンプトを調整して再試行します。

```
1回目: 「記事を書いて」
↓（期待と異なる結果）
2回目: 「初心者向けに、500文字程度の記事を書いて」
↓（まだ改善の余地あり）
3回目: 「プログラミング初心者向けに、変数の概念を説明する500文字程度の記事を書いてください。具体例を2つ含めてください」
```

## よくある失敗パターンと対策

### 失敗パターン1: 曖昧すぎる指示
```
❌ 「何か面白いことを教えて」
✅ 「プログラミングに関する興味深い歴史的エピソードを3つ教えてください」
```

### 失敗パターン2: 情報不足
```
❌ 「コードをデバッグして」
✅ 「以下のPythonコードでIndexErrorが発生します。原因を特定し、修正方法を説明してください。[コードを挿入]」
```

### 失敗パターン3: 矛盾する指示
```
❌ 「詳細に説明してください。ただし簡潔に。」
✅ 「主要なポイント3つに絞って、それぞれ100文字程度で説明してください」
```

### 失敗パターン4: 出力形式の未指定
```
❌ 「データを整理して」
✅ 「以下のデータをMarkdown形式の表として整理してください」
```

## 高度なテクニック

### Chain of Thought（思考の連鎖）
複雑な推論が必要な場合、段階的に考えるよう促します。

```
以下の数学の問題を解いてください。
ステップごとに考え方を説明しながら解答してください：
[問題文]
```

### ゼロショット vs フューショット

**ゼロショット**: 例を示さずに指示のみ
```
次の文章を英語に翻訳してください：「こんにちは」
```

**フューショット**: 例を示してパターンを学習させる
```
以下のパターンで翻訳してください：

日本語: おはようございます
英語: Good morning

日本語: ありがとうございます
英語: Thank you very much

日本語: こんにちは
英語:
```

### プロンプトチェーニング
複雑なタスクを複数のステップに分割します。

```
ステップ1: 以下のテキストから主要なキーワードを5つ抽出してください
ステップ2: 抽出したキーワードを使って要約を作成してください
ステップ3: その要約を英語に翻訳してください
```

## まとめ

効果的なプロンプトを作成するための重要なポイント：

1. **明確に**: 何を求めているかを明確に伝える
2. **具体的に**: 詳細な情報と制約条件を提供する
3. **構造化**: 論理的に整理された形で情報を提示する
4. **文脈を含める**: 背景情報や目的を説明する
5. **例を示す**: 望ましい出力の例を提供する
6. **反復改善**: 結果を見ながらプロンプトを調整する

プロンプトエンジニアリングは実践を通じて上達します。様々なプロンプトを試し、結果を観察し、改善を繰り返すことで、より効果的なプロンプトを作成できるようになります。
